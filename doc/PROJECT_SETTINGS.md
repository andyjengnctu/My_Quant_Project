# 專案設定

## 策略邏輯
1. 單股回測不使用該檔股票自身的歷史績效 filter 作為買入閘門；單股回測只用來驗證核心策略本身對該檔股票的適用性。
2. 單股回測必須完整依技術訊號、盤前掛單、成交／未成交、停損／停利規則產生交易歷程，不可因該檔自身歷史 EV、勝率或交易次數而提前封鎖後續進場。
3. 歷史績效 filter 僅能用於投組層的候選資格、排序、是否掛單等決策；不得回頭影響單股回測本身的交易生成。
4. 投組層使用的歷史績效統計來源，應來自單股回測產生的完整交易結果；不得使用投組自身是否實際買到作為統計母體。
5. 技術訊號、候選資格、是否掛單、是否成交、miss buy、歷史績效統計，必須分層定義，禁止混用。
