## A. 回覆原則
1. AI 註解與使用者註解必須明確區分；只能刪 AI 註解，不可刪使用者註解，嚴禁擅改或刪除原本正確的程式碼。
2. 只提供客觀分析與建議，不奉承。
3. 但需先檢討自已，才可提出合理質疑。
4. 何時侯都要保持禮貌，語氣要客氣。
5. 重點回答，避免重覆說明相同概念。
6. 回答要明確，是就是是、非就是非，避免似乎、機乎、比較好、比較差這種模稜兩可的用語。
7. 當使用者要求「最嚴格檢查」時，必須一次找出所有問題，包含程式、工具、統計口徑、交易限制符合性，並包含全檔靜態審核、py_compile 語法檢查、合成案例等驗證。
8. 如需做動態測試，測試資料(tw_stock_data_vip_reduced)放在專案資料中，程式包中不會放; 如需要外部套件(如optuna)就自行安裝，環境設定寫在requirements/requirements.txt中。
9. 檢查有問題就直接提供修改，一次把"所有"問題都找出，並撤底修好，避免事後再擠出新問題，並嚴格確認沒有退回之前錯的版本。
10. 如果是提供更新的程式碼片段，總是先提供明確要被取代的完整片段(需確認過可直接貼上搜尋就找到)，縮排也必須可直接貼上使用，讓我可以無腦的copy-search-past就完成。
11. 如果是提供更新的程式碼.ZIP檔，只給有改的檔案就好，放在正確的資料匣目錄中，並列出改了哪些檔，讓我可以直接copy-past取代舊檔案。

## B. Coding 原則
1. 單一真理來源：相同邏輯不得在不同地方重複實作。
2. 統計口徑絕對一致：回測、優化、模擬、驗證工具中的成交、未成交、miss buy、EV、勝率、Round-Trip PnL 定義必須完全一致。
3. 避免 magic number；公式與參數需具數學意義與可解釋性。
4. 禁止裸 except，所有異常必須可追蹤。
5. 架構調整不可明顯犧牲效率；若增加未來策略修改或 ML / DRL / LLM 升級複雜度，必須先明確說明。

## C. 交易原則
1. 杜絕未來函數：不可偷看未來資料。
2. 同 K 棒同時觸發停損 / 停利時，一律以最壞停損情況計算。
3. 資金與權益曲線一律以扣除手續費、稅金後的淨值為準。
4. 半倉停利僅視為現金回收，尾倉結算才算完整 Round-Trip PnL。
5. 只能盤前掛單；盤中不得新增、改單、換股。
6. 買入股數上限、總資金占用、最大持倉數、單筆風險曝險，都必須在盤前即可確定。
7. 不得當沖；買入當日不可賣出。
8. 當日賣出回收資金，不得於當日再投入。
9. 若某檔當日未成交，不得於同一交易日盤中自動改掛其他股票。
10. 停利 / 停損僅能對已持有部位預先設定。

## D. 候選 / 延續候選 / Miss buy 定義
1. 當日買入候選資格定義：某檔股票於某交易日，必須同時符合技術買入訊號與歷史績效門檻，才算當日買入候選。
2. 每日買入候選池由當日新取得候選資格之標的與延續候選共同組成。
3. 是否掛單由當日盤前排序、可用資金、持倉名額、風險股數上限共同決定。
4. 延續候選定義：某檔股票只要曾於先前交易日出現技術買入訊號但尚未買入，後續交易日只要仍符合當日延續技術條件、歷史績效門檻與風險限制，即可重新進入候選池參與排序；
5. 某檔前一日未被選到、資金不足、名額不足、或有效掛單未成交，後續交易日只要仍符合當日延續技術條件、歷史績效門檻與風險限制，皆可再次進入候選池參與排序。
6. 延續候選不是特殊通道；不得因曾進入候選池、曾掛單、曾未成交，就繞過當日排序直接保送買進。
7 .miss buy 定義：已於盤前保留資金與名額，且已形成有效限價買單，但盤中未成交。
8. 以下情況都不算 miss buy：排序未選上、資金不足、持倉名額不足、風險股數為 0、先達停損而放棄進場。
9. 候選資格、是否掛單、是否成交、是否 miss buy，必須分開定義，禁止混用。

## E. 策略邏輯
1. 單股回測不使用該檔股票自身的歷史績效 filter 作為買入閘門；單股回測只用來驗證核心策略本身對該檔股票的適用性。
2. 單股回測必須完整依技術訊號、盤前掛單、成交／未成交、停損／停利規則產生交易歷程，不可因該檔自身歷史 EV、勝率或交易次數而提前封鎖後續進場。
3. 歷史績效 filter 僅能用於投組層的候選資格、排序、是否掛單等決策；不得回頭影響單股回測本身的交易生成。
4. 投組層使用的歷史績效統計來源，應來自單股回測產生的完整交易結果；不得使用投組自身是否實際買到作為統計母體。
5. 技術訊號、候選資格、是否掛單、是否成交、miss buy、歷史績效統計，必須分層定義，禁止混用。

## F. 專案特例
1. v16_portfolio_sim.py 自動開瀏覽器暫時允許。
2. 暫時都只使用還原價，不考慮raw。
3. 資料集預設：tools/validate_v16_consistency.py 預設使用 tw_stock_data_vip_reduced；v16_portfolio_sim.py、v16_vip_scanner.py、v16_ml_optimizer2.py 預設使用 tw_stock_data_vip；除 consistency 外不在一般 UI 額外詢問資料集；若需切換，只能透過 CLI `--dataset` 或環境變數覆蓋，且不得把未顯示資料集選單視為問題。
